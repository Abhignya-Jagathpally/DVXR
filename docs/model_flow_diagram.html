DVXR · multimodal prediction · model flow

Predicting glucose & mind-state from EEG + PHR + PPG.

The framework ingests the DVXR lab's own device streams — Galea/EMOTIV brain signals, the wearable personal-health record, and the pulse photoplethysmograph — and fuses them into one set of calibrated clinical predictions, explained without inventing a number.

EEG · BCI Galea · EMOTIV + PHR · wearable iHealth watch — HR/HRV/SpO₂/EDA + PPG · pulse photoplethysmography
01

Device streams in

LSL · time-aligned

The three device streams arrive over Lab Streaming Layer, time-synchronised per subject. Whatever is present is used; the framework degrades gracefully as devices connect or drop.

EEGPHRPPG
02

Per-modality encoders

causal · past-only

Each stream is embedded by an encoder that understands it: a real LaBraM EEG foundation model for the brain signal, and causal physiology features (rolling HR/HRV, pulse morphology) for the wearable PHR and the pulse PPG. No feature reads the future.

EEG → LaBraM embeddingPHR → physiologyPPG → pulse features
03

Per-modality experts

estimate + uncertainty

Every modality becomes an expert that outputs an estimate and its confidence. A modality the model trusts less — noisy EEG, a stale pulse read — contributes less.

04

Availability-aware fusion

abstains if no modality usable

Experts are combined by weighting each device on its current quality and staleness. If none is usable, the model abstains rather than guessing — the safeguard that keeps it honest.

05

Multi-task calibrated heads

split-conformal intervals

The fused representation feeds one head per clinical target, each calibrated so its probability or interval is trustworthy — not just a raw score.

07

Explanation layer

grounded · never predicts

Each prediction is explained in plain language — restating the model's own numbers and drivers. The explainer is gated so it can phrase, never compute: it cannot introduce a value the model didn't produce.
