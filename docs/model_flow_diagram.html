NeuroGlycemic · model flow

How the framework forecasts blood glucose from device signals.

One availability-aware model turns whatever the DVXR devices provide — CGM, the wearable/pulse watch, logged meals — into a calibrated 30-to-120-minute glucose forecast, then explains it without inventing a number. Below is the whole path, signal to explanation.

01

Device streams in

LSL · availability-aware

Live signals arrive over Lab Streaming Layer. Whatever is present is used; nothing is required — the model degrades gracefully as devices drop in and out.

CGMpulse HRmealsEEG*
02

Causal features, per device

past-only

Each stream becomes lags, deltas, rolling mean/SD and slopes computed only from time t or earlier — no peeking at the future it is asked to forecast. This is what keeps the error honest.

03

Per-device experts

Gaussian μ, σ

Every device gets its own expert that outputs a glucose estimate and its uncertainty, conditioned on the forecast horizon. A device the model trusts less contributes less.

04

Meal response kernel

sign-constrained +Δ

A physiologically-shaped carb→glucose response is added to the experts. Meals can only raise future glucose, with a bounded gain — the model can't invent an implausible dip from food.

meals → Δ glucose
05

Availability-aware fusion

abstains if no device usable

Experts are combined by weighting each device on its current quality, staleness and clock certainty. If nothing usable remains, the model abstains rather than guessing — and says so.

06

Residual over persistence

correct “stays flat”

The network predicts a correction to the naive “glucose stays where it is” baseline, shrunk toward it. That inductive bias is why it reliably beats persistence instead of trailing it.

07

Calibrated forecast

PI-95 coverage 0.91

Split-conformal calibration turns the mixture into an honest interval at each horizon — a point value plus a 95% band you can trust.

08

Explanation layer

grounded · never predicts

A deterministic renderer — an LLM-ready, LLM-gated seat — restates the model's own numbers (risk band, top drivers) in plain language. It is validated so it can phrase, never compute: it cannot introduce a value the model didn't produce.

What each device contributes

CGM dominates — remove it and error nearly triples. Meals add ~0.4; the pulse device adds little at 30 min but more by 120 min, and keeps a forecast alive (~90% coverage) when CGM drops out. Every device earns its place, honestly measured.

Why not just a bigger model?

same-split ladder · RMSE @30 min

On raw point accuracy, gradient boosting ties or wins — the causal features are the real win, not depth. The neural model earns its keep through calibrated uncertainty, abstention, and multi-device fusion, and we report that trade-off rather than hide it.

What this is — and isn't

  • Research-grade, not a device. Every output carries validated_for_clinical_use = false; the one remaining gate is prospective external validation.
  • Glucose is forecast from CGM + wearable/pulse + meals — real device streams. EEG (EMOTIV/Galea) is trained today for the mental-health heads (depression AUROC 0.961) and is a wired, honest slot for glucose the moment co-registered EEG+CGM data is captured — not faked.
  • Temporally causal, not causal-inference. No future leakage; contributions are associational attributions, not intervention effects.
  • The explanation is deterministic here — an LLM seat that stays gated to restating model numbers, so it can never hallucinate one.
